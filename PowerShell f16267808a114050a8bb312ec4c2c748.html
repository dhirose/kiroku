PowerShell

特徴

  • PowerShellで作ったスクリプトを実行するには、初期設定が必要
  • コマンドプロンプトと違い、実行結果がオブジェクトで返る
    • コマンドプロンプトはテキストで返る
  • .NET Frameworkのライブラリが使える
    • https://wa3.i-3-i.info/word13559.html
  • COM(Componet Object Model)が使える
  • WMI(Windows Management Instrumentation)が使える

用語集

PowerShellプロバイダー

  • ファイルシステムやレジストリなどのデータ領域を抽象化する仕組み
  • https://learn.microsoft.com/ja-jp/powershell/module/microsoft.powershell.core/about/about_providers?view=powershell-7.4

PowerShellドライブ

  • ハードディスク上のドライブにアクセスするのと同様に、パスを使用してPowerShellプロバイダーにアクセスするための仕組み

コアコマンド

  • PowerShellプロバイダーが使用するために設計されたコマンドレット

UAC

  • 「User Account Control」の略
  • 管理者権限で実行する機能
  • https://wa3.i-3-i.info/word1159.html

WQL

  • WMI用のSQL文

COM

  • 「Component Object Model」の略
  • 異なるアプリケーション間でプログラムを利用するための技術
  • https://e-words.jp/w/COM.html

WMI

  • 「Windows Management Instrumentation」の略
  • https://wa3.i-3-i.info/word14903.html

ADO

  • ActiveX Data Objectの略
  • Microsoft者が開発したデータベースを操作するためのCOM
  • AccessやSQL Serverを操作できる

NuGet

  • オープンソースのパッケージマネージャー
  • https://freelance.shiftinc.jp/column/nuget

PowerShell Core

  • クロスプラットフォーム(Mac OS、Linux OS上でも動作する)を意味する。
  • オープンソースソフトウェア
  • クロスプラットフォームで動作するPowerShell実装。.NET Coreを使って開発されており、複数のプラットフォームで動作。開発は活発で、今後しばらくは新機能の開発などが積極的に進められる見通し。コマンドはpwsh.exeまたはpwsh。
  • 参考：https://news.mynavi.jp/techplus/article/techp3625/

Windows PowerShell

  • WindowsおよびWindows Serverに同梱して出荷されてきた、これまで10年に渡って使われたオリジナルのPowerShell。.NETフレームワークを使って開発されており、Windows専用。開発は終了しており、今後はバグフィックスのみが行われる。コマンドはpowershell.exe。
