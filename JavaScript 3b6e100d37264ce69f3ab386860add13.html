JavaScript

  • リファレンス
    • https://developer.mozilla.org/ja/docs/Web/JavaScript/Reference
  • javascriptの書く場所
    • HTMLの<script>タグの中

メモ

  • javaScriptの文字列は、シングルコーテーション、ダブルクォーテーションのどちらで囲んでもよい。
    • ...が、以下の理由でシングルコーテーションのほうが良い
      • ①HTMLの属性値はダブルクォーテーションが使われるため、それと区別する。
      • ②見やすい（気がする）
  • 命令文の最後に「;」を付ける。
    • 付けなくても動作する命令文もあるが、基本的に付けておいたほうが余計なトラブル回避になる。
  • エスケープ表記
    • https://developer.mozilla.org/ja/docs/Web/JavaScript/Reference/Global_Objects/String#エスケープ表記
  • DOMとは
    • ノードの種類
      • 要素ノード
      • 属性ノード
      • テキストノード
  • 「window.」の記載は省略できる
    • 例）「window.alert('hoge');」⇒「alert('hoge');」
  • コンソール出力
    • console.log(’hoge’)
  • 要素の取得
    • document.querySelector(’#ID’)
  • 定数の宣言
    • const 定数名 = 値;
  • 変数の宣言
    • let 変数名;
    • ※varもあるが、問題があった為letに変わった経緯がある。もうvarは使わないほうがいい
  • イベントの発動
    • 対象.addEventListener(’イベント名’, 動作)
  • 関数の宣言方法
    • const 関数名 = ([引数], [引数]…) ⇒ {}
    • function 関数名() {}
  • テンプレート文字列
    • 「`(バッククォート)」で囲んで、変数やコードは${変数名}でアクセスできる。
    • 例：`カラーコード：${document.querySelector(’#colorPicker’).value}`
  • クラスの付与
    • 要素.classList.add(’クラス名’);
  • クラスの削除
    • 要素.classList.remove(’クラス名’);
  • 要素.insertAdjacentHTML(’挿入場所’, ‘<div></div>’)
    • 要素を挿入(追加)する
    • innerHTMLとの違いは、innerHTMLは要素を置換する。
  • 近年のJavaScriptでは、constを積極的に使う
    • 意図せず値を再代入してしまうミスを防げる
    • 値が再代入されないことがconst宣言を見ればわかる
